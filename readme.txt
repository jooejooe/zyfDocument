每个案件类型都对应一个XXXAjGroupQueryParamVo、XXXXAjGroupQueryResultVo和XXXXAjGroupQueryContext，XXXX为案件简称，比如民事为msaj

XXXAjGroupQueryParamVo：是该类型案件查询的查询参数，默认继承了FyAjGroupQueryParamVo，FyAjGroupQueryParamVo里只有最基础的案件查询字段，如果你要对该类型案件的查询参数进行扩充请在本类型案件的查询参数中新增即可，然后别忘了在fenix的xml中新增该字段查询的sql片段。

XXXXAjGroupQueryResultVo：是该类型案件查询的返回值，继承了AbstractAjGroupQueryResultVO，里面包含了案件类型主表和一些子表，主表的泛型和一些公用的子表比如诉讼参与人都在父类里，如果要新增只有该案件特有的子表请在子类中扩充

XXXXAjGroupQueryContext：XXXXAjGroupQueryContext是在执行案件查询责任链的上下文对象，继承了FyajGroupQueryContext，里面包含了案件编号List、业务类型、输入参数泛型和输出泛型。

扩展： 

1、如果你要对某个类型案件新增一个子表查询，请在XXXXAjGroupQueryResultVo里加入这个子表，然后新建一个command，这个command在spring里的beanname要是这个表的表名。

2.如果你要新增一个主表的返回查询字段请在每个主表的JCDTO里新增即可。
